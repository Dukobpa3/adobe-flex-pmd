1. Add/update some files containing the violation you want to detect
2. Figure out the kind of you rule: Ast-Based (AST stands for Abstract Synthax Tree) or regular expression based.
3. Create a unit-test for this rule depending on its type.
4. Create the rule specifying its base class
5. Write the unit-test
6. Write the actual rule
7. Run the flex-pmd-ruleset test suite
8. Once it succeeds, update or create the ruleset containing the new rule.
9. Describe the new rule in the ruleset. It will be used for the documentation.
10. Run the flex-pmd-core test suite, and update the expected total number of violations in the test-resources.
12. open a command line, change the current directory to <checkOutDirectory> and type : mvn package amd make sure the build succeeds
13. Check the code in if you can, otherwise submit a patch.
